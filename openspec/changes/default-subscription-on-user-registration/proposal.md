## 为什么

当前新用户注册后不会自动获得默认订阅套餐，管理员若希望把某个套餐作为注册福利，需要依赖手工操作或额外流程，既增加运营成本，也让拉新与转化策略难以标准化。现在引入可配置的默认订阅发放能力，可以把注册闭环和订阅体系打通，同时复用现有套餐与额度模型。

## 变更内容

- 增加“新用户注册默认发放订阅套餐”的系统配置，允许管理员启用或关闭该能力，并指定一个默认套餐。
- 在用户注册成功后，根据配置自动为新用户创建默认订阅权益。
- 为默认发放过程增加套餐有效性校验、重复发放保护和失败处理策略，避免影响注册主流程或导致用户收到重复订阅。
- 明确后台配置与用户订阅状态之间的行为边界，使默认发放与现有订阅/计费逻辑兼容。

## 功能 (Capabilities)

### 新增功能
- `registration-default-subscription`: 支持管理员配置新用户注册后自动领取的默认订阅套餐，并在注册成功时为用户发放该套餐。

## 影响

- 受影响系统：用户注册流程、配置中心、订阅套餐与用户订阅管理。
- 受影响代码：注册控制器/服务、订阅服务或模型层、Option 配置读写与管理端配置界面。
- 受影响 API：后台配置读写接口及可能的注册返回行为；不预期引入破坏性 API 变更。
- 受影响运行逻辑：需要定义默认套餐失效、套餐停用、重复注册重试、OAuth/普通注册等路径下的发放策略。
